## Autonomous judgments recorded

### J1. Content-based auto-detection 추가

원래 plan은 filename match (`xdatcar` exact)만. 그러나 CLI 첫 테스트에서
`test-xdatcar-nve` 파일이 dispatch에 안 걸려 single-frame fallback로 빠짐.
`--frame` 플래그가 의미 없게 됨.

**선택**: parseStructureFileTraj fallback에 content marker 검사 추가.
- "Direct configuration=" → parseXdatcarTraj (XDATCAR/Cartesian configuration도 cover).
- "ANIMSTEPS" → parseXsfTraj.

비표준 파일명 (사용자가 XDATCAR 복사본을 임의 이름으로 저장) 모두 처리.

이 변경은 webview에도 영향 없음 (extension은 customEditors selector로
"XDATCAR" 정확매치 필요 — VSCode UX 일관성 유지).

### J2. Frame 0 default + warn 정책

기존 동작 (single PNG 렌더) 보존. trajectory 파일에 `--frame` 미지정이면
frame 0 (= 기존 parseStructureFile.structure와 동일). single-frame 파일에
`--frame N` 지정해도 silent ignore (trajectory.frames.length === 1이라 조건
스킵).

Out-of-range만 warn — 사용자 의도 명시적이지만 잘못 입력 케이스.

### J3. Single-frame fallback 경로 보존

`if (traj.frames.length > 1)` 조건 안에서만 clamp/warn. length === 1일 때
바로 frames[0] 사용. fixed-cell 단일 frame 파일과 trajectory wrapped to
length-1 모두 동일 경로.

### J4. Visual fixture는 frame=3 (중간 frame)

frame 0 = 기존 single-frame 경로와 동일 (baseline 변동 없음). 중간 frame
선택해서 frame 추출 정확성 입증. frame 4도 가능하지만 view=std에서 시각
차이 미세 (Na drift x=0.04 fractional × 5.64Å = 0.225Å, view std는 a*
방향이라 x-axis 변화 거의 안 보임).

xdatcar-frame3 baseline은 frame 0 baseline과 거의 같지만 Na atom이 약간
shift된 상태 — 이후 trajectory 파서/렌더 변경 시 자동 회귀 catching.

### J5. Volumetric은 trajectory에 한정 first frame

trajectory + isosurface 결합은 v0.17 범위 외. parseStructureFileTraj 결과의
`volumetric` 필드는 trajectory와 동등 level에 존재 (per-frame 아님). first
frame에만 의미.

이 commit에서는 변경 없음 — 기존 volumetricJSON 추출이 단일 result.volumetric
사용.

## Known limitations / deferred

1. **Browser per-invocation overhead** — `--frame N` 호출마다 Puppeteer
   런치 (~2초). 17.3.2 `--all-frames`도 동일 (loop 안에서 single instance
   유지는 v0.17.x).
2. **Volumetric per-frame trajectory** — 예: time-varying CHGCAR. v0.17 범위
   외.
3. **--frame range syntax** (`--frame 5-10`) — `--all-frames` + 후처리로
   대체 가능. v0.17.x.
4. **1-indexed alias** (`--frame +1`) — 0-indexed가 표준이지만 사용자 입장에서
   친화적이지 않을 수 있음. v0.17.x에서 옵션 추가 가능.
