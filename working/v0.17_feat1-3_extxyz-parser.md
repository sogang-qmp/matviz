## Autonomous judgments recorded

### J1. parseXyzFrame 추출 — DRY 원칙

기존 `parseXyz`는 single-frame을 직접 파싱. 같은 로직을 `parseXyzTraj`에서
반복하는 대신 frame parser 헬퍼로 추출 → 두 entry 모두 같은 로직 공유.
이것이 v0.17 dual API 패턴(parseStructureFile + parseStructureFileTraj)의
parser-internal 정착.

### J2. Fixed-cell 자동 감지 + REF collapse

ASE는 fixed-cell trajectory를 export할 때 매 frame 같은 Lattice 헤더 작성.
파싱 결과는 frame별로 다른 lattice array 객체. 이를 그대로 두면 renderer
.setFrame이 매 frame "lattice changed" 처리 → 비효율.

**선택**: 파싱 후 모든 lattice array가 bit-equal이면 frames[0].lattice를
모두에 share (REF collapse) → renderer 최적화 활용. 비교 비용 N×9 (N≤수천
frame), 한 번 비용.

### J3. PBC 검출 정책

`Lattice="..."` 가 있으면 pbc=[true,true,true] 디폴트 (기존 parseXyz 동작).
명시 `pbc="T T T"` 있으면 그것 사용. ASE 표준과 일치.

### J4. Properties 컬럼 파싱은 첫 컷에 미지원

ASE는 `Properties=species:S:1:pos:R:3:forces:R:3:magmoms:R:3` 등 임의
컬럼 가능. 첫 컷에서는 species + pos만 활용 (앞 4 토큰). forces/magmoms 등
추가 컬럼은 v0.17.x에서 처리 (per-frame magnetic moment 등 고급 시각화에
연결).

### J5. Plain XYZ regression test

기존 fixtures/water.xyz는 plain XYZ (헤더 없음, 1-frame). `parseXyzTraj`가
이 케이스도 정확히 처리하는지 명시 검증. plain → 1-frame trajectory wrap.

### J6. Trailing junk 안전 처리

`parseXyzTraj` walk loop에서 다음 line이 "양의 정수 atom count"가 아니면
break. 파일 끝 trailing whitespace, 주석 등이 frame 추가 트리거 안 함.
