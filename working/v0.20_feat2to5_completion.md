## 20.2 — `detectSymmetry()` post-parser pass

- `src/parsers/index.ts`: added `withSymmetry()` post-pass that runs
  spglib only when (a) spglib is initialized and (b) the parser hadn't
  already populated a non-`P1` `spaceGroup`. CIF wins on conflict.
- Both `parseStructureFile` and `parseStructureFileTraj` now route
  through it. Trajectory policy: detect on frame 0 only, broadcast the
  result to all subsequent frames (locked decision A).
- `CrystalStructure` gained `spaceGroupNumber` and `hallNumber`
  optional fields.

### Bug fixes caught during implementation

1. **Cartesian → fractional transpose bug** (in `spglibWasm.ts`):
   the original wrapper used `inv * cart` where it should have been
   `inv^T * cart` because the lattice is stored with basis vectors as
   ROWS. Caught by graphene fixture (was reporting P2/m instead of
   P6/mmm). Fixed by transposing the index access in the multiplication.
2. **CIF space-group whitespace** (in `cifParser.ts`): some CIF files
   ship `_symmetry_space_group_name_H-M 'F m -3 m'` with internal
   spaces. Now stripped via `.replace(/\s+/g, '')` to match the
   compact form returned by spglib. Single source of truth: the
   info pill renders the same canonical string regardless of source.
- `src/extension.ts`: added `initSpglibSync(fs.readFileSync(...))`
  call at activation. Wraps in try/catch so init failure is non-fatal
  (falls back to legacy P1 behavior).
- `scripts/render.ts`: same init pattern at CLI top-level.

| Fixture | Expected | Got |
|---|---|---|
| `silicon.poscar` | Fd-3m (#227) | ✅ Fd-3m (#227) |
| `nacl.cif` | Fm-3m (CIF deferred) | ✅ Fm-3m (#?, CIF) |
| `bcc-fe-fm.xsf` | Im-3m (#229) | ✅ Im-3m (#229) |
| `bcc-cr-afm.xsf` | Im-3m (parent) | ✅ Im-3m (#229) — non-magnetic spglib treats AFM Cr as identical species, expected |

All match the physically expected answer for the given atomic
coordinates. The MAPbBr3 case is particularly telling: spglib correctly
detects that the methylammonium dipole pointing along x breaks the
inorganic Pm-3m to monoclinic Pm.

## 20.3 — Info pill correct space group on non-CIF

- `renderer.getStructureInfo()` line 1153 already had `|| 'P1'` as a
  fallback. Comment clarified: post-v0.20, the fallback only triggers
  on (a) spglib init failure or (b) `detectSymmetry` throwing on a
  degenerate cell. No code change otherwise.
- The webview never loads WASM — it consumes the pre-computed
  `spaceGroup` from the structure payload posted by the extension host.

## 20.4 — CLI `--info` flag

- `RenderOptions` gained `info: boolean` and `infoJson: boolean`.
- `parseArgs`:
  - `--info` and `--json` (the latter implies `--info`).
  - Output-default skip: when in `--info`-only mode (no `-o`),
    `parseArgs` no longer auto-fills `opts.output`, so `render()` can
    early-return without spinning up Puppeteer.
- `printStructureInfo()`: computes lattice parameters, volume,
  formula, crystal system from the space-group number, and emits
  either a 5–6 line text summary or a structured JSON object
  (file/formula/atoms/species/lattice/volume/space-group/hall/system).
- Help text + `skills/matviz-render/SKILL.md` flag table + recipe
  + `README.md` flag table all updated in sync (PostToolUse hook
  reminder honoured).
