## Autonomous judgments recorded

### J1. `_visible: false` phase는 rebuild에서 스킵

비저블 토글은 SecondaryPhasesGroup에서 mesh를 빼는 것보다, rebuild 시 해당
phase entry 자체를 스킵하는 게 단순. `(phase as any)._visible` lazy field —
타입 정의 공식화는 v0.17.x.

### J2. removePhase가 phase[0] 삭제 시 자동 clearComparison

comparisonSecondaryPhase가 phase[0] reference. 삭제 시 dangling. 자동
해제로 일관성 유지. v0.17.1 clearPhases와 동일 패턴.

### J3. addPhase 흐름 — webview 버튼 → addPhaseRequest → extension 명령

webview는 file system 접근 불가 → 버튼 클릭 시 webview→extension 메시지
`addPhaseRequest`. extension가 기존 `matviz.addPhase` 명령 호출 (file picker
열림). extension가 parse + addPhase 메시지로 webview에 전달.

대안: extension가 직접 file picker 호출 코드 inline. 하지만 명령 재사용이
DRY + 명령 팔레트 사용자 경로와 동일 동작 보장.

### J4. 토스트는 vscode.window.showWarningMessage

`showErrorMessage`/`showInformationMessage` 대신 warning. compareToPhase
실패는 user error (no secondary phase added) 또는 perf 제한 — error도 info도
아닌 warning이 적절.

### J5. Space 키 input focus 가드

input/textarea/contentEditable focus 시 Space는 default behavior (값 변경)
유지. 그 외에서만 play/pause 토글. UX 안전 패턴.

### J6. Speed slider 0.25×~2× 범위

너무 빠르면 (10×) 시각적 인식 불가. 너무 느리면 (0.05×) 사용자가 직접 scrub
하는 게 빠름. 0.25~2 합리적 범위. 사용자 요청 시 확장 가능.

### J7. Loop on/off default on

MD trajectory 시각화 표준은 무한 루프 — 사용자가 멈추거나 scrub. once-only
는 분석 (특정 transition 끝에서 멈추기) 케이스. default on이 일반적.

### J8. Frame input 변경은 'change' 이벤트 (Enter / blur)

'input' 이벤트는 매 키 입력 (1 → 12 → 123 → ...) → 잘못된 frame jump.
'change'는 confirmed value (Enter or blur) → 단일 jump. Up/down arrow도
'change' 트리거.
