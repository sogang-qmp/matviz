# v0.18.0 Features 18.3 + 18.4 — Mode rail + Floating toolbar (V2 glass)

Bundled because both are pure CSS chrome restyles in `media/styles.css` and
share the same V2 token vocabulary. No HTML or JS changes.

## 18.3 — Mode rail glass

- `#mode-bar` now spans full canvas height (`top: 0; bottom: 0`) per V2,
  glass-tokenized (`--v2-glass-1`, `backdrop-filter: blur(--v2-blur-sm)`).
- `.mode-btn` slightly larger (32×32, was 30×30), V2 hover (`--v2-surf-1`)
  and active states (`--v2-surf-accent` + `--v2-line-focus` border).
- `.mode-sep` swapped to `--v2-line`.
- `z-index: 13` so the rail sits above the floating toolbar (z-index 12).

## 18.4 — Floating centered toolbar

- `#top-bar` swapped from full-width edge-attached to **centered floating
  glass card** (`top: 12; left: 50%; transform: translateX(-50%)`). Content-
  sized (`display: inline-flex`), with a `max-width` that subtracts mode rail
  + side panel + 48px breathing room so it never overlaps either.
- Container queries (`@container topbar`) **dropped** — they were sized
  against full-width. The toolbar now overflows horizontally on narrow
  canvases (scroll bar hidden via `::-webkit-scrollbar { display: none }`).
- `.bar-btn` resized 34×32 → 30×28 (V2 spec is denser), hover `--v2-surf-1`,
  active `--v2-surf-accent` + `--v2-line-focus` border.
- `.bar-sep` thinner (18px tall, was 28px), uses `--v2-line-strong`.
- `.axis-btn` tightened to 26px min-width (a/b/c are short labels).

## Verify

- `npm run build` — clean
- Manual: open any fixture; toolbar floats centered at the top with frosted
  glass, mode rail runs full-height on left, both with subtle border lines.
