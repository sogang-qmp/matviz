## Autonomous judgments recorded

### J1. Inline jacobiSym3 vs symEigen.ts module reuse

`scripts/render.ts`의 inline HTML은 string template이라 import 불가.
`src/webview/math/symEigen.ts` 코드를 string에 그대로 inline.

대안 (`render-helpers.js`로 export → import) 가능성도 있지만 16.3 CLI port
J2 결정과 동일 — 두-소스 유지 부담은 hook reminder + commit 메시지의 명시적
참조로 관리. 코드 동일성을 강조하기 위해 주석에 "matches
src/webview/math/symEigen.ts" 명시.

### J2. Ellipsoid path 우선 (16.2 partial보다 위)

Webview src/webview/renderer.ts buildAtoms와 동일한 우선순위:
1. ellipsoidIdxSet 먼저 빌드 (aniso 있으면 ellipsoid)
2. partialIdxSet은 ellipsoidIdxSet에 없는 atom만
3. 일반 atom 그룹은 둘 다 제외

Aniso + occupancy<1 사이트 (드물지만 가능) → ellipsoid로 렌더, opacity 무시.
Webview J3과 동일 결정 — transparent ellipsoid는 v0.16.x deferred.

### J3. Right-handed frame guard

CLI 코드에 webview의 `det(R) < 0 → last column flip` 가드 그대로 inline
포팅. Eigenvector는 sign-ambiguous → 4×4 매트릭스의 마지막 축 부호를 뒤집어
우향 frame 보장 → mesh normal 일관, 광원 정확.

`scale * (det < 0 ? -r2 : r2)` 한 줄로 처리.

### J4. χ²₃ table 하드코드

webview EllipsoidRenderer와 동일 값:
- CONTOUR_50 = sqrt(2.366)
- CONTOUR_90 = sqrt(6.251)

erfc 라이브러리 도입 회피. 사용자가 0.5/0.9 외 컨투어 원하면 별도 옵션 추가
(현재 미지원).

### J5. View=a fixture (ellipsoid 형태 명료)

Test-aniso.cif Ca는 z축 prolate. view=a는 z축이 screen-horizontal로
투영되므로 ellipsoid의 anisotropy가 명확히 보임 (가로로 길쭉한 모양). view=c는
z축이 카메라축이라 prolate 효과가 안 보임 (그냥 원).

→ Recipes도 동일 패턴 (anisotropic axis가 화면 평면 내에 있도록 view 선택).

## Known limitations / deferred

1. **Symmetry-op rotation R·U·Rᵀ 미적용** — webview와 동일 (cifParser
   1/3 J2 carry-over). 비-P1 + off-diagonal U에서 잘못된 방향.
2. **Transparent ellipsoid (aniso + occupancy<1)** — v0.16.x.
3. **Custom χ²₃ contour 미지원** — 0.5/0.9만. 사용자 요청 시 confidence
   level → χ²₃ inverse CDF lookup 추가.
4. **Ellipsoid impostor 미지원 (CLI도 Phong-only이라 동일)** — webview와
   동일 결정 (uniform-radius 가정 충돌).
