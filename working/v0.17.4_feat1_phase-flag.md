## Autonomous judgments recorded

### J1. Two call sites missed initially → fix mid-implementation

처음에 `replace_all`로 `generateStructureHTML(opts, structureJSON, volumetricJSON)`
2 occurrence 모두 phasesJSON 추가하려 했지만, --all-frames loop의 들여쓰기
차이로 1개만 매칭됐고 single-frame call site는 누락. 첫 visual 테스트에서
phase가 안 보여 발견. 명시적으로 두 번째 call site도 phasesJSON 전달.

**교훈**: replace_all 의도에 의존하지 말고 grep으로 모든 call site 검증.
다음 sub-commit (17.4.2)에서도 같은 패턴 적용 시 양쪽 모두 명시 검증.

### J2. Visual fixture는 silicon.poscar as phase

처음 시도한 test-occupancy.cif (4Å cell, atoms at fractional 0/0.5)는
NaCl(5.64Å) corner와 거의 같은 위치 → transparent overlay가 opaque primary
뒤에 가려서 시각 차이 미세. 회귀 catching에 부적합 (baseline이 phase 없는
buld와 거의 동일하면 phase render 회귀 시 catching 못 함).

silicon.poscar (Si at fractional 0/0.25 in 5.43Å cell)는 cartesian
(1.36, 1.36, 1.36)에 atom이 있어 NaCl 비-corner 위치 → 회색 transparent 원자
명확히 가시. 회귀 안전.

### J3. Phase 파일은 single-frame parse (parseStructureFile)

multi-frame trajectory를 phase로 지정하면 첫 frame만 사용. v0.17.x에서
`--phase file.cif:frame=N` syntax 추가 가능. 첫 컷 단순화.

### J4. Phase opacity/offset hardcode

`PHASE_OPACITY = 0.5`, `PHASE_OFFSET = [0, 0, 0]` (cell-overlap mode).
사용자 명시 옵션 (`--phase file.cif:opacity=0.3:offset=2,0,0`)은 syntax
복잡, 첫 컷에서 deferred. 사용자 워크플로우 핵심 케이스 (relaxed vs
unrelaxed in same cell)는 default 그대로 충분.

### J5. depthWrite:false + renderOrder:1 (v0.17.2 carry-over)

Stacked transparent secondary atoms가 primary opaque 뒤에 정확하게 blend.
v0.17.2 webview 패턴 그대로.

### J6. Inline HTML 페이즈 데이터 — OPTS 외 별도 변수

`phases`는 OPTS object에 묶지 않고 별도 const로 inline (`const phases =
${phasesJSON};`). 이유: phases는 큰 array (atoms 100s), OPTS는 작은 flag
모음 — semantic separation + JSON 가독성.

generateStructureHTML 시그니처에 4번째 param 추가 (`phasesJSON: string =
'[]'`). default empty array 유지 → test mode (no input)와 phase 없는 경우
모두 안전.
