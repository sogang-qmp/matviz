# v0.18.0 Feature 18.8 — Measure HUD overlay

Top-right glass HUD that surfaces the current measurement (distance, angle,
dihedral) with V2-spec hero typography, atom-pair card, Δ readouts, and a
copy-to-clipboard button. Visible only in measure mode.

## HTML — `crystalEditorProvider.ts`

`<div id="measure-hud" class="hidden">` after the info pill, containing:
- `.measure-head` — `.measure-kinds` segmented switcher (Distance / Angle /
  Dihedral) + `.measure-x` close button (uses `ICON.close`).
- `.measure-value` — `#measure-num` (gradient text) + `#measure-unit`.
- `.measure-pair` — atom card grid; populated dynamically with N atoms
  separated by `.measure-connector` arrows.
- `.measure-delta` — Δ fractional + Δ cartesian rows; visible only for
  distance measurements.
- `.measure-foot` — context-aware hint + Copy button.

## CSS — `media/styles.css` appended

Lifted verbatim from `v2.css:437–636` with `.v2m*` → `.measure-*` rename:
- 340px wide, glass-3 background, focus-blue border, `--v2-sh-3` shadow.
- Hero number 34px, weight 700, gradient white→#b8b8b8 via
  background-clip: text. Unit 20px in `--v2-fg-2`.
- Pair grid varies columns by `data-count`: 2 → `1fr auto 1fr`,
  3 → 5 columns, 4 → 7 columns.
- Atom swatch 22×22 circle with inset highlight; symbol 15px bold,
  index 11px tabular-nums.
- Connector: short blue lines + `↔` glyph in focus-blue.
- Δ rows: 13px tabular-nums values right-aligned.

## JS — `main.ts`

### State
- `measureHistory: PickedAtom[]` — bounded to last 4 picks, mirroring the
  renderer's `selectedAtoms` cycle (renderer clears at length ≥ 4 after
  emitting a dihedral).
- `lastMeasurement: { type, value, atoms } | null` — the most recent
  measurement payload from `renderer.setMeasurementCallback`.
- `measurePendingClear: boolean` — set after a dihedral fires; the next
  atom-select wipes history so a new cycle starts clean.

### Wiring
- `setMode()` shows the HUD on `'measure'`, hides on `'navigate'`, and
  resets state on transition.
- `setAtomSelectCallback`: on hit, push to history (or wipe first if
  `pendingClear`); on miss, wipe. In navigate mode, atom-select is
  ignored by the HUD.
- `setMeasurementCallback`: store `lastMeasurement`; if dihedral, set
  `pendingClear`; re-render HUD.
- `renderMeasureHud()`: single source of truth for HUD content. Kind
  switcher reflects current measurement type, with Angle / Dihedral
  disabled until enough atoms are picked. Δ rows visible only when
  type is distance.

### Buttons
- **Copy** — `navigator.clipboard.writeText` of a tab-separated readout
  (`distance Cu#3 Cu#5\t2.5471 A`). Disabled when no measurement.
- **✕** — calls `renderer.clearMeasurements()` and resets HUD state.
- **Esc** key: piggy-backs on the existing Escape handler — clears the
  renderer + HUD state when measure mode is active.

### Kind switcher behavior
The buttons are **informational** in this version, not functional —
matviz's renderer determines measurement type from `selectedAtoms.length`
(2 / 3 / 4). The active button reflects what was last computed, and the
disabled state of Angle/Dihedral previews what's reachable from the
current pick count. A future version could make clicks actively
switch which measurement to lock onto, but that's out of scope for v0.18.

## Verify

- `npx tsc --noEmit` — clean
- `npm run build` — clean
- Manual:
  - Switch to measure mode (◯ / ↔ in mode rail). HUD appears top-right
    with em-dash placeholder.
  - Click an atom → "Click a 2nd atom for distance" hint, single-atom
    card.
  - Click a 2nd → distance value + atom-pair card + Δ rows; Angle button
    enables.
  - Click a 3rd → angle in degrees, 3-atom card, Δ rows hide.
  - Click a 4th → dihedral, 4-atom card; the next click restarts.
  - Copy puts the measurement on the clipboard. ✕ clears.
  - Esc clears + resets the HUD; switching back to navigate hides it.
