# 14.5 — 성능 예산 감사

## 스트레스 픽스처

`test/fixtures/stress-20k.poscar` — 다이아몬드 원시 2원자 × 20³ 슈퍼셀 = **16 000원자** (20k 타깃 근접, 파일 크기 관리 차원에서 20³ 선택). 슈퍼셀 UI로 추가 확장 가능.

## 타깃 (플랜에서 확정)

- 16 000–20 000원자 구조:
  - Draw call ≤ 200
  - 유휴 FPS = 0 프레임 (requestRender 게이트)
  - GC 후 힙 ≤ 200 MB
- 1 000원자 구조:
  - 회전 중 FPS ≥ 55 (SwiftShader)
  - Draw call ≤ 50

## 구조적 보장

코드 수준에서 이미 충족된 제약:

1. **Draw call**: 원자는 원소별 `InstancedMesh` 1개 — 탄소 단일 원소 구조 16 000원자는 draw call 1회. 결합은 >5000원자에서 자동 스킵 (`renderer.detectBonds` line 1472). 14.6에서 도입된 `forceBonds` 옵션을 꺼 둔 기본 경로에서 결합 draw call 0.
2. **유휴 FPS**: `renderFrame`은 `requestRender` 게이트로만 트리거 — OrbitControls 변경, state 변경 시에만. 정지 시 rAF 0.
3. **축 인디케이터 누수**: 14.7 `AxisIndicator.disposeContents()` 가 geometry/material/texture 재빌드마다 해제. 팔레트 토글 / 슈퍼셀 변경 루프에서 `renderer.info.memory` 증가 없음 (예상).

## 라이브 측정이 필요한 항목

헤드리스 SwiftShader로 FPS / 힙을 재는 것은 별도 하네스가 필요. 권장 측정 프로토콜 (사용자 수동):

1. `code test/fixtures/stress-20k.poscar` 오픈 → 결합 스킵 힌트 노출 확인.
2. 크롬 DevTools `Performance` 녹화 중 회전 드래그 3초.
3. `Memory` → Heap snapshot → GC 후 힙 사이즈 확인.
4. 콘솔에서 `renderer.info.render.calls`, `renderer.info.memory.geometries/textures` 평가.

이 수치는 다음 릴리스 (v0.15) 시 회귀 비교 기준으로 사용.

## 결정

- 구조적으로 타깃 만족 경로가 확보됨. 중대한 최적화 작업 없이 v0.14 릴리스 진행.
- 라이브 측정은 사용자 하드웨어에서 수행 후 회귀 발생 시에만 추가 작업.
