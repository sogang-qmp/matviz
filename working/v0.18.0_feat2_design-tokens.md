# v0.18.0 Feature 18.2 — V2 design tokens

Adds the V2 design-token block to `:root` in `media/styles.css`. No
visual change yet — these are vars consumed by Features 18.3–18.8.

## Tokens added

- **Glass layering**: `--v2-glass-1/2/3` (0.72 / 0.86 / 0.94 alpha),
  `--v2-blur` (16px), `--v2-blur-sm` (10px)
- **Lines**: `--v2-line` (0.06), `--v2-line-strong` (0.10),
  `--v2-line-focus` (rgba 0,127,212,0.55 — VSCode focus blue)
- **On-panel surfaces**: `--v2-surf-0/1/2` (0.03 / 0.06 / 0.10),
  `--v2-surf-accent` (focus-blue tinted)
- **Shadows**: `--v2-sh-1/2/3` (escalating depth + outline)
- **Radii**: `--v2-r-sm/r/lg` (4 / 6 / 8 px)
- **Type scale**: `--v2-t-caps` (10) → `--v2-t-num` (34)
- **Foreground aliases**: `--v2-fg-1/2/3` mapped from VSCode tokens

## Side panel width bump

`--side-panel-w` raised from 200px → 248px (V2 spec). `--layout-offset-left`
still references it; the offset layout will be removed entirely in Feature
18.5, so this is a transitional state where offset mode shifts a few pixels.
Acceptable for one commit.

## Verify

- `npx tsc --noEmit` — clean
- `npm run build` — clean
- No runtime change. Token block is consumed by subsequent features.
