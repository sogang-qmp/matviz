## Autonomous judgments recorded

### J1. Triple-intersection 알고리즘 vs Sutherland-Hodgman clipping

Plan §결정 게이트 #7는 "Sutherland-Hodgman 변형 ~80 LOC"를 채택. 그러나 실구현
중 더 단순한 대안 발견:

**선택: Triple-intersection method.**
- 모든 planes triple (i,j,k)를 enumerate
- 각 triple의 3×3 linear system 풀기 → intersection point
- 그 점이 모든 *나머지* planes 만족하면 polytope vertex
- ConvexGeometry로 모든 vertex의 hull → 삼각화

**근거:**
- Sutherland-Hodgman 변형은 polytope 토폴로지 (vertex/edge/face) 명시 유지
  필요 → 복잡 (~150 LOC + 디버그).
- Triple-intersection은 O(n^4) (C(n,3) × n 검사) 이지만 n ≤ 14 (큐빅 + 8
  octahedral) → ~1500 ops, 한 순간.
- ConvexGeometry는 v0.15.2에서 이미 도입 (다면체) → 인프라 재사용.
- Numerical robustness: triple-intersection은 mathematical하게 더 깔끔 (vertex
  = active constraint set, 정의에 가까움).

코드 ~180 LOC (Sutherland-Hodgman ~150 보다 약간 큼이지만, 대부분 doc/comment).

### J2. Bounding box fallback

User input이 unbounded region 허용 시 (예: 단일 plane), naive 알고리즘은 무한
polytope 시도. **Fallback: 사용자 planes에 bounding box 6 planes 추가**.

- Bbox size = `max(γ) × 4`로 충분히 크게 → bounded valid Wulff에서는 bbox
  vertices가 모든 user planes 외부 → discarded.
- Under-constrained user input은 bbox로 visible (사용자 디버그 도움).
- Trade-off: 정확한 Wulff는 user가 fully bounded planes 입력 가정.

테스트 2 (single plane)에서 fallback 동작 확인.

### J3. Symmetry expansion 미적용

Cubic 결정에서 (1,0,0)는 6개 symmetry-equivalent {100} face를 가짐. 사용자가
하나만 입력해도 자동 확장하면 편의성 향상. 그러나:

**선택: 미구현. 사용자가 모든 face explicit 입력.**

근거:
- Symmetry expansion은 space group 추론 필요 (현재 스코프 외).
- 명령 팔레트 default 값에 cube 6면 미리 채워서 체험 부담 완화.
- 16.x: lattice의 cubic-ness 휴리스틱 또는 명시 space group 입력 후 expansion.

### J4. 사이드 패널 UI deferred

Plan §16.4는 사이드 패널 toggle + opacity 슬라이더 명시. 그러나:

근거:

LOC + UI 작업량 절감.

node에서 import 시도 → 정상 동작 확인 (test-wulff.js Au cuboctahedron 케이스
44 triangles 통과). QuickHull 알고리즘 부분은 pure math.

esbuild가 three.js 전체를 bundle inline → node에서 자체 완결.

### J7. Distance/normal 단위

Lattice 입력은 Å. Reciprocal `a* = b×c/det(...)`는 1/Å. Plane 거리 γ는 사용자
"임의 단위" — Wulff theorem은 face-distance가 surface energy에 *비례*만
요구하므로 γ의 절대값은 visualization scale만 결정.

코드: γ를 그대로 plane distance로 사용. polytope 크기는 cell parameter 비교
시각적으로 적절 (Au 4.08 Å lattice + γ=1.0 → 변 ~2 Å Wulff). 너무 작거나 크면
사용자가 γ 조정.
