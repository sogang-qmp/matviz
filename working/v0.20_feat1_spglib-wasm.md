# v0.20.1 — Build/integrate spglib WASM artifact (completion log)

**Status**: complete (2026-04-27).
**Commit**: pending — first feature of v0.20, awaiting user check-in
before continuing to 20.2.

## What landed

- `npm install --save @spglib/moyo-wasm@^0.7.9` — official Spglib org
  Rust port, MIT/Apache-2.0 dual license, TS declarations included.
- `src/shared/spglibWasm.ts` — TS wrapper with `initSpglib(url)` /
  `initSpglibSync(buf)` / `detectSymmetry(lattice, cartPos, species)`.
  Cartesian → fractional conversion done internally so callers stay
  unaware of the moyo input convention. HM symbol is normalized to
  compact form ("Im-3m" not "I m -3 m") to match matviz's CIF parser
  output. Returns `null` on initialization not-yet-called or on moyo
  throwing (degenerate lattice, NaN positions).
- `esbuild.mjs` — `copySpglibWasm()` step copies
  `node_modules/@spglib/moyo-wasm/moyo_wasm_bg.wasm` to both `dist/`
  (CLI loads via `fs.readFileSync`) and `media/` (webview loads via
  `webview.asWebviewUri`). Hard-fails build if raw size > 700 KB.

## Gotchas captured

1. **HM symbol whitespace**: moyo emits `"I m -3 m"`; matviz's CIF
   parser stores `"Im-3m"`. Normalized in the wrapper via
   `.replace(/\s+/g, '')`.
2. **Setting parameter**: the third arg to `analyze_cell` is a string;
   `'Spglib'` (uppercase S) is the correct value. Confirmed via smoke
   test; documented in inline JSDoc.
3. **Input is fractional**: moyo's `MoyoCell.positions` expects
   fractional coords. matviz stores Cartesian internally. The wrapper
   does the lattice-inverse multiplication once per `detectSymmetry`
   call — cheap (3×3 matrix invert + N matvecs).
4. **Unknown species → Z=0**: `Z_MAP` covers elements 1..103. Any
   custom symbol (deuterium "D", custom labels) maps to 0 which moyo
   treats as a valid distinct atom type — symmetry result still
   reflects the intended occupation as long as identical labels share
   the same Z.
