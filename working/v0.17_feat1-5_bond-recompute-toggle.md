## Autonomous judgments recorded

### J1. cachedBonds inherit 전략

가장 간단한 perf 전략: 첫 frame bond를 그대로 모든 frame에 사용. 트레이드오프:
- 안정 MD trajectory (작은 atom motion): bond 시각적으로 정확.
- 큰 motion (phase transition, melting): bond가 부정확하지만 backbone 시각화
  여전히 의미 있음.

대안 (atom 거리 변화 임계값으로 selective recomputation)은 복잡도 증가, 첫
컷에서 보류.

### J2. rebuild(resetCamera, skipBondsRecompute) 시그니처 확장

기존 6개 호출 사이트는 `rebuild()` 또는 `rebuild(false)`. positional bool
보존 + skipBondsRecompute는 두 번째 옵션 추가. 기존 호출 사이트 수정 0.

대안 (options object)은 더 깔끔하지만 시그니처 break — 호출 사이트 6개 모두
수정 필요. 자율 진행 중 risk 최소화 위해 positional 유지.

### J3. 5k auto-disable + checkbox.disabled

UI에 큰 N에서는 사용자가 토글 못 켜도록 명시 disable. trajectory section
visibility 갱신 시점에 atom count 검사. 토글 켜진 상태에서 큰 trajectory
열면 자동 off + UI 갱신.

5k threshold는 v0.14 bond skip hint 임계값과 동일 — 일관성. 더 정밀한 perf
benchmark는 17.x.

### J4. setFrame이 latticeMode='fixed' 시 cell 재빌드 — 그대로 두기

원래 plan: `latticeMode='fixed'` 시 setFrame에서 cell wireframe 재빌드 스킵
가능. 그러나 `rebuild()`가 `disposeResources()` → `buildUnitCell()` 묶음으로
처리. 분리하면 dispose 로직이 cellGroup만 살리는 분기 필요 — 복잡도 증가.

**선택**: 일단 cell 매번 재빌드 (latticeMode 무관). 1k×1k frames에서 perf
충분하면 OK; 부족하면 v0.17.x에서 cell-skip 분기 추가.

### J5. CachedBonds inherit + style switch interaction

사용자가 trajectory 재생 중 display style을 ball-and-stick → space-filling으로
바꾸면 setDisplayStyle → rebuild() 호출. 이 rebuild는 default skipBondsRecompute=
false → 첫 frame bond 재계산 (현재 frame이 아닌 frame 0의 cached bonds
재계산). frame N 시점에서는 frame N의 atom 위치로 detectBonds — 그 결과를
이후 frame들이 계속 사용 (현재 frame이 새 frame-0-equivalent로 됨).

이는 의도된 동작이지만 UX적으로 약간 혼란 가능. v0.17.x에서 정리 가능 (style
switch 시 첫 frame으로 jump 또는 명시 cache). 첫 컷에서는 문서화로 처리.
