# v0.13 Feature 1 — Headless CLI Renderer

**Date**: 2026-04-16
**Files**: `scripts/render.ts`, `dist/render.js`, `esbuild.mjs`, `package.json`

## Summary

Completed 13.1–13.8 in a single iteration. A Node.js CLI (`node dist/render.js`)
now renders crystal structures to PNG images using Puppeteer + SwiftShader.

## What ran

### 13.1 — Prototype (headless-gl attempt)

- Installed `gl`, `pngjs`.
- `gl(1,1)` returned `null` on vesper — server has Mesa but no X display; `gl`
  requires working GL context for headless-gl's native addon.
- **Decision**: switched to Puppeteer fallback as anticipated in the plan.

### 13.1b — Puppeteer prototype

- Installed `puppeteer` (~400 MB Chromium bundled).
- Initial attempts failed:
  1. `networkidle0` + CDN `three.module.js`: timed out (no network? CSP?).
  2. `file://` + `setContent`: CORS blocked ES module imports from `origin='null'`.
  3. `--disable-gpu`: WebGL context creation failed.
  4. `--use-gl=swiftshader` alone: context was created then immediately lost
     (Chromium deprecation warning on software WebGL without opt-in flag).
- **Final flag set that works**:
  ```
  --no-sandbox --disable-setuid-sandbox
  --enable-unsafe-swiftshader --enable-webgl
  --allow-file-access-from-files
  ```
- Test scene (red sphere on dark bg) renders correctly.

### 13.2–13.5 — Structure rendering (combined)

Self-contained HTML template inlined into `scripts/render.ts`. The template:
- Imports Three.js 0.170 via `file://` from `node_modules/three/build/three.module.js`.
- Contains an element data table (trimmed subset of `elements-data.ts`) so the
  HTML is fully self-contained — no `postMessage` bridge needed.
- Renders atoms (InstancedMesh per element), bicolor bond cylinders, cell
  wireframe (LineSegments), boundary atoms (fractional wrap + face/edge/corner
  images), camera (ortho + axis views), all 4 display styles.

### 13.6 — CLI interface

`parseArgs()` handles all options documented in `plans/v0.13_cli-renderer.md`.
No external dependency (e.g. `commander`) — straight `process.argv` parsing to
keep the bundle lean.

### 13.7 — Addon features (completed)

All four addon features now implemented in the HTML template:

- **`--labels`**: canvas-based sprite labels per atom (element symbol, white text
  on translucent black rounded-rect). Texture cache per element symbol so a 1000-atom
  scene with 5 elements only allocates 5 textures.
- **`--polyhedra`**: coordination polyhedra for atoms with ≥4 bonded neighbors.
  Convex hull approximation via outward-facing triangle enumeration. Transparent
  faces (opacity 0.4, DoubleSide) + edge outlines (LineSegments). Silicon
  tetrahedra render cleanly; perovskite shows the Ca/Sr coordination overload
  inherent to the bond-cutoff-based detection (matches GUI behavior).
- **`--iso <level>`**: marching cubes isosurface for volumetric data (Cube/XSF/
  CHGCAR). Both positive and negative lobes rendered (blue / red, VESTA convention).
  Marching cubes module reused from `src/webview/marchingCubes.ts` via a separate
  `dist/render-helpers.js` ESM bundle loaded into the HTML page.
- **`--plane h,k,l`**: lattice plane via Miller indices. Plane normal computed
  from reciprocal lattice; orange semi-transparent square (1.5x cell size).

The marching cubes algorithm and its 256-entry tables are bundled separately
(`dist/render-helpers.js`) rather than inlined in the HTML, to keep the
inline template manageable.

## Decisions locked in

- **Rendering backend**: Puppeteer (headless Chromium + SwiftShader), not
  headless-gl. Reason: works out of the box on headless Linux without X/GL,
  full WebGL2 support matches Three.js 0.170 requirements.
- **HTML template**: inlined in `scripts/render.ts` as template literal. Reason:
  CLI must be self-contained; no runtime HTML loading path.
- **Element data**: trimmed duplicate of `elements-data.ts` inside the HTML
  template. Reason: avoiding cross-bundle imports in the browser context.
- **Bond detection**: simple O(N²) inside HTML template (not spatial hash).
  Reason: CLI renders are typically small structures; if N>5000 becomes
  common we'll port the spatial hash.

## Issues / follow-ups

- **Deprecation**: `--enable-unsafe-swiftshader` is Chromium's name for
  "allow software WebGL for non-browser contexts." Chromium may change this
  flag name in future. If Puppeteer breaks after a Chromium upgrade, check
  the flag name first.
- **GPU stall warnings** during ReadPixels: cosmetic (printed to stderr),
  the render is correct. Could suppress with `--disable-logging` or pipe
  stderr. Left as-is for now — useful when debugging.
- **13.7 addon features** (labels, polyhedra, isosurface, lattice plane):
  code hooks exist but HTML template doesn't render them yet. Low priority —
  core rendering covers the common case. Revisit on demand.
