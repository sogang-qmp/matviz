# 14.1 — 사이드 패널 레이아웃 모드

## 변경 요약

- 기본값을 **offset** 모드로 전환 (VESTA 스타일). 사이드 패널이 캔버스를 밀어내 원자가 패널 뒤로 가려지지 않음.
- **overlay** 경로는 보존 — Layout 섹션의 토글로 언제든 복귀 가능. `body.layout-offset` 클래스 부재 시 기존 풀폭 캔버스 동작을 그대로 유지.
- 선택한 모드는 `vscode.setState`에 저장되어 탭을 닫았다 열어도 복원.

## 건드린 파일

- `media/styles.css` — `--mode-bar-w` / `--side-panel-w` / `--layout-offset-left` CSS vars, `body.layout-offset #canvas` 규칙, `.layout-toggle-row` 스타일.
- `src/editor/crystalEditorProvider.ts` — 사이드 패널에 Layout 섹션 (Offset / Overlay 버튼) 추가.
- `src/webview/main.ts` — `applyLayoutMode()`, `updateLayoutOffsetVar()`, `setLayoutMode()` (저장 포함), 패널 접힘/리사이즈 시 offset CSS var 재계산, 영속 상태에 `layoutMode` 추가.
- `src/webview/renderer.ts` — `onResize()`가 `canvas.clientWidth/Height`를 우선 사용. ResizeObserver가 캔버스 + 부모를 관찰해 offset/overlay 모두에서 올바르게 리사이즈.

## 통과 기준 대비

| 기준 | 결과 |
|------|------|
| 캔버스가 패널 뒤로 가려지지 않음 | offset 모드에서 캔버스가 패널 오른쪽에서 시작 ✅ |
| 모드 선택 가능 | Layout 섹션 Offset/Overlay 두 버튼 ✅ |
| 드래그 리사이즈 유지 | `#panel-resize` 핸들 동작 유지, offset CSS var가 드래그 중 갱신 ✅ |

## 수동 검증 필요

- [ ] offset 모드에서 원자가 패널 뒤에 숨지 않는지
- [ ] overlay 모드로 토글하면 캔버스가 풀폭으로 복귀하는지
- [ ] 패널 접기/펴기가 두 모드에서 캔버스 크기를 맞게 갱신하는지
- [ ] 패널 드래그 리사이즈가 offset 모드에서 캔버스 왼쪽 끝을 따라가는지
- [ ] 탭을 닫았다 열면 선택한 레이아웃 모드가 복원되는지
