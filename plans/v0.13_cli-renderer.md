# v0.13 — Headless CLI Renderer

## Goal

matviz GUI와 동등한 정적 렌더링을 커맨드라인에서 수행하는 Node.js 스크립트. Claude 등 AI 에이전트가 구조 파일을 자동으로 시각화·내보내기할 수 있게 한다.

## Scope

### 포함 (CLI로 포팅할 기능)

**핵심 — GUI와 1:1 대응되는 정적 렌더링 기능**:

| 기능 | GUI 대응 | CLI 옵션 | 우선순위 |
|------|----------|----------|---------|
| 파일 파싱 (CIF, POSCAR, XSF, XYZ, PDB, Cube, CHGCAR, QE, FHI-aims) | 자동 감지 | 입력 파일 경로 | 필수 |
| 디스플레이 스타일 (ball-and-stick, space-filling, stick, wireframe) | 드롭다운 | `--style` | 필수 |
| 슈퍼셀 확장 | 입력 필드 | `--supercell 2,2,2` | 필수 |
| 카메라 모드 (ortho/perspective) | 토글 버튼 | `--camera ortho\|persp` | 필수 |
| 뷰 방향 (a, b, c, a*, b*, c*, standard) | 축 버튼 | `--view a\|b\|c\|a*\|b*\|c*\|std` | 필수 |
| 임의 회전 | 마우스/키보드 | `--rotate 30,0,45` (x,y,z 도) | 필수 |
| Bond 표시/숨기기 | 체크박스 | `--no-bonds` | 필수 |
| Boundary atoms | 체크박스 | `--no-boundary` | 필수 |
| Unit cell 와이어프레임 | 체크박스 | `--no-cell` | 필수 |
| 출력 PNG | 스크린샷 버튼 | `-o output.png` | 필수 |
| 해상도 | 2x 고정 | `--width 1920 --height 1080` | 필수 |
| 색상 팔레트 (dark/light) | 토글 버튼 | `--palette dark\|light` | 필수 |
| 배경색 | 테마 자동 | `--bg "#1e1e1e"` 또는 `--bg transparent` | 필수 |

**부가 — 있으면 유용한 기능**:

| 기능 | CLI 옵션 | 우선순위 |
|------|----------|---------|
| Atom labels | `--labels` | 중간 |
| Polyhedra | `--polyhedra` | 중간 |
| Isosurface (volumetric data) | `--iso 0.05` | 중간 |
| Lattice plane | `--plane 1,1,1` | 낮음 |
| 원소별 색상 오버라이드 | `--color Na=#ff0000` | 낮음 |
| 원소별 반지름 오버라이드 | `--radius Na=1.2` | 낮음 |
| Bond cutoff 오버라이드 | `--bond-cutoff Na-Cl=3.0` | 낮음 |
| 축 인디케이터 | `--axis-indicator` | 낮음 |
| CIF/POSCAR 내보내기 | `--export cif` | 낮음 (별도 스크립트로 분리 가능) |

### 제외 (CLI에 불필요한 인터랙티브 기능)

| 기능 | 제외 이유 |
|------|-----------|
| 원자 피킹/선택 | 마우스 클릭 기반 — headless에서 의미 없음 |
| 거리/각도/이면각 측정 | 피킹 의존 — 필요 시 좌표 계산으로 대체 (렌더��� 불필요) |
| 실시간 회전/줌 (OrbitControls) | 인터랙티브 전용 |
| 사이드 패널/UI 오버레이 | GUI 전용 |
| 세션 상태 저장/복원 | CLI는 stateless |
| 드래그 리사이즈 | GUI 전용 |
| 모드 전환 (navigate/measure) | 인터랙티브 전용 |

## 기술적 결정

### headless-gl vs Puppeteer

| | headless-gl (`gl` npm) | Puppeteer |
|---|---|---|
| 의존성 | C++ addon, 빌드 필요 | Chromium 다운로드 (~400MB) |
| Three.js 호환 | WebGL1 subset, 일부 shader 제한 | 완전한 WebGL2 |
| 속도 | 빠름 (네이티브 GL) | 느림 (브라우저 기동) |
| 안정성 | shader 호환 이슈 가능 | GUI와 동일 렌더링 보장 |

**권장: headless-gl 우선 시도**. MeshPhongMaterial + InstancedMesh 수준은 WebGL1으로 충분. shader 호환 문제 발생 시 Puppeteer fallback.

### 코드 재사용 전략

- **파서**: `src/parsers/` 그대로 import (이미 Node.js 호환)
- **원소 데이터**: `src/parsers/elements.ts` 사용 (webview용 `elements-data.ts` 대신)
- **렌더러**: `CrystalRenderer`는 DOM/canvas에 결합되어 직접 재사용 불가. 핵심 로직(원자 배치, 본드 감지, 슈퍼셀 확장, 바운더리)을 추출하거나 headless용 경량 렌더러를 새로 작성.
