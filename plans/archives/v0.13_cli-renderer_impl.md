# v0.13 — CLI Renderer Implementation Spec

## 13.1 — headless-gl + Three.js 프로토타입

### Setup

```bash
npm install gl pngjs three @types/three
```

### 프로토타입 스크립트 구조

`scripts/render.js` (또는 `scripts/render.ts` → esbuild로 별도 빌드):

```
1. gl 패키지로 headless WebGL context 생성 (width x height)
2. Three.js WebGLRenderer에 context 주입: new WebGLRenderer({ context, ... })
3. 간단한 장면 (빨간 구 하나) 렌더링
4. gl.readPixels → pngjs → PNG 파일 저장
```

### Pass criterion

`node scripts/render.js --test` → 빨간 구가 보이는 test.png 생성.

### 알려진 리스크

- `gl` 패키지가 서버 환경(vesper)에서 빌드 가능해야 함 (mesa/libGL 필요)
- Three.js r150+ 는 WebGL2 기능 사용 — `gl`이 WebGL1만 지원하면 fallback 필요
- **Locked-in value**: {{pending — 13.1 결과에 따라 결정}}

## 13.2 — 파서 통합 + 원자 렌더링

### 파서 재사용

```typescript
import { parseStructureFile } from '../src/parsers/index';
```

esbuild로 `scripts/render.ts`를 번들할 때 파서를 inline — `vscode` 모듈 import가 없으므로 그대로 빌드 가능.

### 원자 장면 구성

GUI `CrystalRenderer.buildScene()` → `expandSupercell()` → `renderAtoms()` 흐름을 경량 함수로 재구현:

1. `expandSupercell(structure, [na, nb, nc])` → species[], positions[]
2. `addBoundaryAtoms(...)` (boundary 옵션 시)
3. 원소별 `InstancedMesh` 생성 (SphereGeometry + MeshPhongMaterial)
4. 원소 색상: `getElement(symbol)` from `src/parsers/elements.ts` + palette 변환

### 빌드 설정

`esbuild.mjs`에 세 번째 엔트리 추가:

```javascript
// CLI renderer bundle
esbuild.build({
  entryPoints: ['scripts/render.ts'],
  bundle: true,
  platform: 'node',
  outfile: 'dist/render.js',
  external: ['gl', 'pngjs'],
  format: 'cjs',
});
```

### Pass criterion

`node dist/render.js nacl.cif -o nacl.png` → NaCl 원자가 올바른 색상/크기로 보이는 PNG.

### Data location

출력 PNG: 사용자 지정 경로 또는 기본값 `{input_stem}.png`.

## 13.3 — 본드 + 셀 와이어프레임 + 바운더리

### 본드 감지

`detectBonds()` 로직을 CLI 렌더러에 포팅:
- spatial hashing (cell-list)
- per-pair cutoff from `autoPopulateBondParams()`
- periodic boundary consideration

### 본드 렌더링

bicolor cylinder (반반 색상) — GUI와 동일한 `CylinderGeometry` + `InstancedMesh`.

### 셀 와이어프레임

`LineSegments` + `LineBasicMaterial` for:
- outer cell boundary (실선)
- supercell internal slices (대시)

### 바운더리 원자

`generateBoundaryAtoms()` 로직 포팅 — fractional coord wrap + face/edge/corner images.

### Pass criterion

NaCl 2x2x2: 본드 + 셀 라인 + 바운더리 원자 모두 보이는 PNG. GUI 스크린샷과 비교.

## 13.4 — 카메라 뷰 + 회전 + 줌

### 카메라 설정

```typescript
// Orthographic (기본)
const frustum = computeFrustumFromBBox(boundingBox);
const camera = new OrthographicCamera(-frustum*aspect/2, frustum*aspect/2, frustum/2, -frustum/2, 0.1, 500);

// Perspective
const camera = new PerspectiveCamera(50, aspect, 0.1, 500);
```

### 뷰 방향 계산

GUI의 `viewAlongAxis()` / `standardOrientation()` 로직 재사용:
- a/b/c: 격자 벡터 방향에서 바라봄
- a*/b*/c*: 역격자 벡터 방향
- std: c축 위, a* 방향에서 바라봄

### 임의 회전

`--rotate x,y,z` (도 단위) → Euler 회전을 카메라 position에 적용.

### Fit to view

장면 bounding box로부터 자동 카메라 거리/frustum 계산.

### Pass criterion

6축 뷰 각각에 대해 nacl.cif PNG 생성 → GUI 축 버튼 결과와 일치.

## 13.5 — 디스플레이 스타일 4종

각 스타일별 원자/본드 렌더링 파라미터:

| Style | Atom radius | Bond | Bond radius |
|-------|------------|------|-------------|
| ball-and-stick | displayRadius (0.30–0.65) | bicolor cylinder | 0.08 |
| space-filling | vdwRadius (1.2–2.1) | 없음 | — |
| stick | 0.15 (bonded) / displayRadius (unbonded) | cylinder | 0.08 |
| wireframe | displayRadius | LineSegments | — |

### Pass criterion

4종 스타일 × nacl.cif PNG → 각각 GUI와 시각적으로 일치.

## 13.6 — CLI 인터페이스 + 옵션 파싱

### 사용법

```
Usage: node dist/render.js <input> [options]

Arguments:
  input                Structure file path (CIF, POSCAR, XSF, XYZ, PDB, Cube, etc.)

Options:
  -o, --output <path>  Output PNG path (default: {input_stem}.png)
  --width <n>          Image width in pixels (default: 1920)
  --height <n>         Image height in pixels (default: 1080)
  --style <s>          Display style: ball-and-stick|space-filling|stick|wireframe (default: ball-and-stick)
  --camera <c>         Camera mode: ortho|persp (default: ortho)
  --view <v>           View direction: a|b|c|a*|b*|c*|std (default: std)
  --rotate <x,y,z>    Additional rotation in degrees (default: 0,0,0)
  --supercell <a,b,c>  Supercell expansion (default: 1,1,1)
  --palette <p>        Color palette: dark|light (default: dark)
  --bg <color>         Background color hex or "transparent" (default: #1e1e1e)
  --no-bonds           Hide bonds
  --no-boundary        Hide boundary atoms
  --no-cell            Hide cell wireframe
  --labels             Show atom labels
  --polyhedra          Show coordination polyhedra
  --iso <level>        Isosurface level (volumetric data only)
  --plane <h,k,l>     Add lattice plane
  -h, --help           Show help
```

### 옵션 파싱

의존성 최소화: `process.argv` 직접 파싱 또는 가벼운 라이브러리 (`commander` 또는 직접 구현).

### Pass criterion

`--help` 출력 정상, 잘못된 옵션에 에러 메시지, 모든 옵션 조합 동작.

## 13.7 — 부가 기능

### Labels

Canvas 기반 sprite 텍스처 — headless에서는 `node-canvas` (`canvas` npm) 또는 Three.js `CanvasTexture` with offscreen canvas. `gl` 환경에서 sprite가 동작하는지 13.1에서 확인 필요.

### Polyhedra

`buildPolyhedra()` 로직 포팅 — convex hull mesh + transparent material + edge outlines.

### Isosurface

`marchingCubes()` 로직 포팅 — volumetric data 파싱 후 BufferGeometry 생성.

### Pass criterion

각 기능 별도 PNG로 확인. LiF-polaron-isolevel.xsf에서 isosurface 동작.
