## 19.1 — Split-pane: text editor + 3D view

### Architecture (option B baseline)

- `CustomReadonlyEditorProvider<CrystalDocument>` stays as-is.
- `crystalEditorProvider.ts` `resolveCustomEditor` registers
  `vscode.workspace.onDidChangeTextDocument` filtering to the open
  document's URI; debounce 250 ms.
- On a debounced change event, the host reparses the new text content,
  runs the v0.20 `withSymmetry` post-pass, and posts a `structureUpdate`
  message into the webview.
- Webview's existing `loadStructure` message handler accepts the new
  payload, calls `renderer.loadStructure(...)` which already supports
  re-loading without a full webview rebuild.

### Files

- Modified: `src/editor/crystalEditorProvider.ts` — add the change-event
  subscription + debounce inside `resolveCustomEditor`.
- New (optional): `src/editor/companionEditor.ts` — a thin command
  registered as `matviz.openWithText` that opens both the text editor
  and the matviz view side-by-side via `vscode.window.showTextDocument`
  + `vscode.commands.executeCommand('vscode.openWith', uri,
  'matviz.crystalViewer', vscode.ViewColumn.Beside)`.
- Modified: `package.json` — add `matviz.openWithText` command +
  optional `editor/title` menu entry.

### Recipe

```ts
// In resolveCustomEditor, after the initial parse + post:
const debounceMs = 250;
let parseSeq = 0;
let timer: NodeJS.Timeout | null = null;

const sub = vscode.workspace.onDidChangeTextDocument((e) => {
  if (e.document.uri.toString() !== document.uri.toString()) return;
  if (timer) clearTimeout(timer);
  const seq = ++parseSeq;
  timer = setTimeout(() => {
    try {
      const text = e.document.getText();
      const parsed = parseStructureFileTraj(text, filename);
      // Drop result if a newer keystroke kicked in during reparse:
      if (seq !== parseSeq) return;
      webviewPanel.webview.postMessage({ type: 'structureUpdate', ... });
    } catch (err) {
      webviewPanel.webview.postMessage({ type: 'parseError', message: String(err) });
    }
  }, debounceMs);
});
webviewPanel.onDidDispose(() => sub.dispose());
```

### `parseError` UI

Webview adds a small "(stale: parse error)" badge to the info pill when
a `parseError` arrives. Clears on next successful parse.

### Wiring

- `crystalEditorProvider.ts` reads `vscode.workspace.getConfiguration('matviz')`
  during `resolveCustomEditor` and passes the resolved defaults into
  the webview as part of the existing init message.
- Webview's renderer init applies the defaults *only when* localStorage
  has no saved state for the current URI. localStorage takes precedence
  for return visits.

### Settings ↔ localStorage interaction (documented)

> Per-file persistence (localStorage / `vscode.setState`) wins after the
> first visit. Settings provide initial defaults for fresh opens or
> sessions where the cache was cleared.

## 19.3 — Property-panel undo/redo

### Stack design

- Webview-only state. Not persisted across sessions (intentional —
  Ctrl+Z behavior follows VSCode pattern of session-scoped undo).
- Stack depth: 50.
- Tracked operations:
  - element color override
  - element radius override
  - element visibility toggle
  - per-pair bond min/max edit
  - per-pair bond enable toggle

### Files

- New: `src/webview/undoStack.ts` — generic undo/redo stack with
  `record(undo, redo)`, `undo()`, `redo()`, depth-cap eviction.
- Modified: `src/webview/main.ts` — wrap each property-panel listener
  with a `recordChange(beforeState, afterState)` call. Bind `Ctrl+Z`
  / `Ctrl+Shift+Z` keydown handlers (skip when an input is focused —
  reuse the existing digit-shortcut input-skip logic).

### Recipe

```ts
// undoStack.ts
export class UndoStack<T> {
  private undos: Array<{ undo: () => void; redo: () => void }> = [];
  private redos: typeof this.undos = [];
  private cap: number;
  constructor(cap = 50) { this.cap = cap; }
  record(undo: () => void, redo: () => void) {
    this.undos.push({ undo, redo });
    if (this.undos.length > this.cap) this.undos.shift();
    this.redos = [];
  }
  undo() { const e = this.undos.pop(); if (!e) return; e.undo(); this.redos.push(e); }
  redo() { const e = this.redos.pop(); if (!e) return; e.redo(); this.undos.push(e); }
}
```

```ts
// In main.ts color picker handler (sketch):
colorInput.addEventListener('change', () => {
  const before = renderer.getElementColor(el);
  const after = colorInput.value;
  renderer.setElementColor(el, after);
  undoStack.record(
    () => renderer.setElementColor(el, before),
    () => renderer.setElementColor(el, after),
  );
});
```

### Keybinding

Default `Ctrl+Z` / `Cmd+Z`: caught only when no input/textarea/contenteditable
is focused (so VSCode's text-buffer undo still works in the side-by-side
text pane). Webview keydown listener reuses the same skip-detection used
for digit shortcuts (1–4).
