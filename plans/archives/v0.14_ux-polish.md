# v0.14 — UX 다듬기

**목표:** 일상 사용 편의성 개선. 어떤 에디터 폭에서도 뷰어가 쓸 만하게, 상태가 재오픈 후에도 살아남게, "왜 아무 일도 안 일어나지?" 순간에 대한 답을 발견 가능하게 만든다 (대형 구조 결합 스킵, 단축키 찾기 등).

**상태:** 킥오프 초안 — 착수 전 승인 대기.

## 범위

`Plan.md`의 v0.14 표를 의존성 기준으로 재정렬하고 v0.13.1 릴리스를 반영해 수정함:

|---|------|-----------|
| 14.1 | 사이드 패널 레이아웃 모드 — 캔버스 offset vs overlay | 캔버스가 절대 가려지지 않음; 모드 선택 가능; 드래그 리사이즈 유지 |
| 14.2 | 반응형 툴바 브레이크포인트 (compact / normal / wide) | 400–2000 px 전 범위에서 모든 버튼 접근 가능; 컨트롤이 숨는 wrap 없음 |
| 14.3 | 단축키 도움말 오버레이 (`?` 키) | 최초 사용자가 한눈에 모든 단축키 파악; 닫기 가능 |

## 피처 간 의존성

- 14.7 (AxisIndicator 추출)은 13.1.5에서 도입한 renderer 레지스트리를 건드리므로 **맨 마지막**에 배치해 중간 리팩터링 소용돌이 방지.
- 14.4 (상태 영속성)는 14.1 (레이아웃 모드 자체가 저장 대상)에 의존.
- 14.5 (성능 예산)은 14.1–14.4가 만든 최종 상태를 상대로 측정해야 함 — 마지막 직전.
- 14.2 (반응형 툴바)는 독립.
- 14.3 (도움말 오버레이)은 독립. v0.13.1에서 tooltip은 이미 도입됐으므로 그 위에 "발견" 레이어를 쌓는 구조.
- 14.6 (결합 UX)는 독립.

**제안 순서:** 14.1 → 14.2 → 14.3 → 14.6 → 14.4 → 14.7 → 14.5.

## 원래 Plan.md 대비 재정렬 근거

- **14.3**을 14.4보다 앞으로: 도움말 오버레이는 자체 완결형으로 즉시 사용자 가치 전달. 영속성은 여러 표면을 건드려서 레이아웃/패널 확정 후가 안전.
- **14.6**을 14.4 앞으로: "Compute anyway" 버튼이 새 저장 선호를 만들므로 영속성에 한 번에 꿰는 편이 수월.
- **14.7**을 끝에서 두 번째로: 사용자 눈에 보이지 않는 리팩터링은 기능 작업을 막지 말아야 함.
- **14.5** 마지막: 표면이 안 정해진 상태에서 최적화는 무의미.

## 착수 전 확정 필요 질문

1. **14.1 기본 모드:** overlay (현재) vs offset (캔버스 축소)? offset이 VESTA식에 가까움. **제안: 기본값 offset, 토글 노출, 선택 저장.**
2. **14.3 오버레이 형태:** 모달 다이얼로그 vs fly-out? **제안: 모달**, 키보드로 닫기 가능, 카테고리별 단축키 표.
3. **14.6 "Compute anyway" 노출:** 5000원자 하드 캡 유지 vs 항상 버튼 표시? **제안: 자동 스킵은 유지**, 스킵됐을 때만 힌트 + 우회 버튼 노출.
4. **14.5 예산 타깃:** 현 `Plan.md`는 "draw call < 100, GPU 유휴 0 프레임, 메모리 < 100 MB". 1000원자 기준은 `InstancedMesh`로 이미 달성됨. **제안: 스트레스 기준을 20 000원자 (2500원자 원시셀 × 슈퍼셀)로 재설정.**

구체 파일/코드 레시피는 `v0.14_ux-polish_impl.md` 참조.
